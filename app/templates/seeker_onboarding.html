Seeker Onboarding

From install to first confirmed booking, designed to feel fast and trustworthy.

The seeker path is built to delay permissions until they are useful, surface trust before payment, and preserve momentum from the first search to the first booking.

Activation Milestone

What counts as activated?

A seeker is activated when the first booking reaches confirmed state within the same session or within 24 hours of the first booking attempt.

Trust Signals

What the seeker should see early.

Verified providers, transparent pricing, ratings, next available slots, and protected in-app chat should appear before checkout.

What Can Wait

Deferred until there is clear intent.

Saved addresses, notification permission, payment save, profile photo, and marketing consent can all be deferred until later.

Screen Flow

Recommended step-by-step seeker journey

  1. Welcome and role choice.Route the user correctly and set trust expectations from the first screen.
  2. Mobile and OTP.Use secure login before any irreversible step or private booking state.
  3. Name and lightweight setup.Collect only what is required for receipts, support, and search relevance.
  4. Search and compare.Let seekers browse with filters, compare providers, and understand trust before committing.
  5. Booking or quote mode.Fixed work moves into slot booking, while custom jobs move into structured quote requests.
  6. Checkout.Ask for address, payment, and promos only at the point where the user already has booking intent.
  7. Post-booking operations.Move directly into private chat, notifications, live tracking, dispute support, and review prompts.

Permission strategy

Location is requested only when the user wants nearby results. Notifications are requested after search or booking intent, not on app launch.

48-hour recovery

If no booking happens, re-engage with last-search recall, verified-provider prompts, and a first-booking promo only for qualified low-intent seekers.

Support promise

If something fails, seekers should get exact recovery copy, preserved drafts, and a clear next step instead of a dead end.
